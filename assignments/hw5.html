Aim

This homework will get you more familiar with settings bundles.
Task

Create another preference specifier for LanguageList, a “Multi Value” element.

  • The element should allow the user to choose from a list of language families1.
  • The title should reflect that.
  • The possible values of the preference can be whatever you want, but one of them should be “Germanic”, and that should be the default value.
  • You should reflect the preference in the application by the ListController’s title. You don’t have to change the list of languages based on the preference, unless you’re ambitious and into this sort of thing.

1 Technically, the Germanic languages are not a language family in and of themselves. A language family is a “maximal set” of related languages; that is, there are no languages outside the family that are demonstrably related to any languages in the family. This is not true of Germanic languages. They are a subfamily of the Indo-European language family, which contains a wide variety of other subfamilies: any spoken in Europe except Basque, Hungarian and Finnish, as well as the vast multitude of Indic languages descended from Sanskrit.
Hints

Settings bundles aren’t a Cocoa class, so the documentation for them is not where most of the docs you’ve used so far are. The Settings Application Schema Reference should be very useful. You can also look at the Application Preferences chapter of the iPhone Application Programming Guide.

Looking at the docs for NSUserDefaults could be helpful too.
