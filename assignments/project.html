Final Project: Word Search

The final project will be a word search game. This is a common pencil-and-paper game, with a square grid of letters, in which several words are embedded. You’re given a list of the words, and you have to find and circle each of them in the grid. Words can run horizontally, vertically and diagonally, in either direction. This kind of interaction lends itself well to the touch screen: when you find a word, you can just drag your finger over it.

Puzzle Generation

Generating new word search puzzles is nontrivial, and that’s not the point of this project, so I’ve implemented the logic for you. Download the code above and drop it into your Xcode project. Look at the sample project and the comments in PuzzleGenerator.h to see how to use it. Also, make sure to read the comments at the top of PuzzleGenerator.m for important information about the algorithm and its limitations.

You’re free to modify the puzzle generation code as much as you like. However, make sure to say you did so in your documentation (see the “Submission and grading” section) so I know to look at your changes.

No
collaboration

There are only three acceptable kinds of code in a project submission: code you wrote yourself, code I wrote, and code that you adapted from Apple documentation or sample code.

In particular, of course, this means you are not allowed to share code among yourselves. You can certainly share advice, but not code.

You’re free to read any sources you want (Cocoa help sites, etc.) but I don’t want you copying code from them. If you’re not sure if something constitutes copying, ask me.
Submission
and grading

Submission is through LabFusion.

Note: include a readme documenting your general approach, any major UI design decisions you made, any extra features you made, etc.

Grading is out of 15 total points.

  • 6 points for the UI: well-designed, well-thought out and attractive UIs get more points.
  • 6 points for code quality: consistent and reasonable coding style, correct memory management, good use of APIs (for example, you’d lose credit for reimplementing functionality that’s already in Cocoa), and good separation of functionality into classes and MVC layers.
  • 3 points for polish: any additional features you added, how appropriate and well-executed they are.
